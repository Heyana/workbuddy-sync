## Memory Block

**工作背景**
用户是全栈开发者，兼具游戏UI设计背景，桌面存有FPS游戏UI分析PPT及国风射击游戏UI设计文档。技术栈涵盖 Electron + Go + SQLite 架构及 Flutter 开发。核心项目包括：Flutter 资产管理 App（WorkBuddy-Space/asset_manager），具备多账号 WebDAV 文件级增量同步；Electron 媒体管理器（electron-media-manager），采用 Go 后端（HTTP 端口 23347）与内置 SQLite，已从 Prisma 迁移；osvtoolbox 项目，涉及基于 RTX 4070 GPU 的 ffmpeg 视频编码逻辑优化。同时，用户正在开发名为 'dots' 的 Flutter 个人生活追踪 App（使用 shadcn_flutter + Wind + LucideIcons 技术栈）、PureRaw 照片筛选/评分工具（采用 liquid_glass_widgets 实现 macOS 26 Tahoe Liquid Glass UI 风格），以及 ZUI v2 (原 z_vue3) Vue 3 组件库。ZUI v2 技术栈为 Vite 6 + Vue 3.5 + TS 5.6 + Tailwind v4 + shadcn-vue，包含 35 个组件（如 ZButton, ZTable, ZVirtualList 等）及 toast/dialog/drawer 等 API，构建产物为 dist/zui.js (ES, gzip 37.8KB) 和 dist/zui.umd.cjs (UMD, gzip 29.4KB)，预设代码为 `a2QcktPc`（禁止解码）。此外，用户使用 WorkBuddy（腾讯产品），桌面也有飞书、钉钉等办公软件，并与韩语环境相关（桌面有韩文办公套件文件，涉及헤즈이/Hezii）。用户还负责维护 linktwins.com 域名下的服务（如 xishu-online.api, wechat-file-manager），并频繁进行网络请求追踪与服务器 IP 目的地检查。

**个人背景**
用户风格直接、务实，偏好在提供项目背景后获取简洁的反馈与编译结果。遵循“代码审查 → 性能分析 → 架构文档”的工作流，要求 AI 阅读源代码文件、审查 git diff 以诊断问题，并基于原始日志提供根因分析与具体修复方案的结构化输出（含表格、代码块）。UI 设计偏好 Naive UI 组件库与简洁商务风格，在 Flutter 项目中则严格使用 shadcn_flutter 并避免 material/forui 组件；迭代时习惯通过截图和浏览器 DevTools CSS 片段指出细节问题，且偏好内联 SVG 图标而非 Emoji。在 Flutter 开发中 enforce 严格规则：禁用 forui/material，使用 shadcn toast，优先使用 div/wdiv 而非 Expanded/Column，且查询必须使用 proper where 子句而非 '1=1' 原始 SQL。用户期望代码变更完成后执行 commit 并 push，同时要求每次代码修改完成后需自动更新记忆、spec 等文档完成留痕。偏好将项目记忆文件存储在项目文件夹内以提升便携性，并参考 `D:\hxy\github\test_wind_1_1` 项目的 alias 设置。

**当前关注**
正聚焦 Flutter 资产记账 App 的数据库可靠性与同步架构升级，此前在目录迁移中遭遇数据丢失，已将同步方案转为多账号 WebDAV 文件级增量同步，并计划新增数据库路径可视化显示以增强数据位置确认。同时，正在排查 Electron 媒体管理器中 GORM + SQLite 的慢 SQL 写入问题（>200ms），并致力于 osvtoolbox 项目的 ffmpeg 编码性能优化，充分利用 RTX 4070 硬件能力。Flutter 习惯/事件追踪 App 'dots' 正按 Timeline→Stats→Settings→Category CRUD→Daily Goals→Dark Mode→Export→Reminders 路线图推进，当前重点解决 Timeline 滚动加载时出现的 'Incorrect use of ParentDataWidget' 错误。此外，PureRaw 照片筛选工具 UI 开发已启动，聚焦 liquid_glass_widgets 实现的 Liquid Glass 风格界面与 AI 评分功能；ZUI v2 组件库开发正在进行，重点推进 35 个组件的实现与 API 封装。

**近期动态**
- 完成 Flutter 项目架构调整：移除 OneDrive 支持，新增多账号 WebDAV 同步，采用文件级增量同步替代 ZIP 全包同步。
- 排查并修复 Flutter App 数据库迁移后的数据丢失问题，计划增加数据库路径可视化显示以增强系统状态确认。
- 推进 Electron 媒体管理器后端性能优化，重点解决 GORM + SQLite 写入延迟超过 200ms 的瓶颈。
- 启动 osvtoolbox 项目开发，正针对 RTX 4070 GPU 优化 ffmpeg 视频编码逻辑（osvtoolbox.cpp）。
- 明确 Flutter 习惯/事件追踪 App 'dots' 的技术栈与开发规范，开始按路线图推进功能开发，并 enforce 严格的组件与布局使用规则。
- 正在进行 Flutter 'dots' 项目的 Timeline 功能开发，重点实现滚动加载并解决 ParentDataWidget 相关的布局报错。
- 启动 PureRaw 照片筛选工具项目（D:\hxy\github\pureraw_flutter），选定 liquid_glass_widgets 库以实现 macOS 26 Tahoe Liquid Glass 设计风格。
- 推进 ZUI v2 (原 z_vue3) Vue 组件库开发，确立最新技术栈（Vite 6, Vue 3.5, TS 5.6, Tailwind v4），完成 35 个组件定义、API 规划及构建配置（ES/gzip 37.8KB）。
- 持续通过截图与 CSS 片段迭代各 App UI，偏好使用 Naive UI 及 shadcn_flutter 组件库，并要求每次改完后自动更新记忆和 spec 留痕。
- 维护 linktwins.com 相关后台服务，频繁进行网络层诊断与 IP 追踪。
